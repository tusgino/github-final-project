# Contributing

Thank you for your interest in this project.

All contributions, bug reports, bug fixes, documentation improvements, enhancements, and ideas are welcome.

## How to Contribute

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-idea`)
3. Make your changes
4. Commit with a clear message
5. Push to your fork and open a Pull Request

## Bug Reports

Please include:
- Steps to reproduce
- Expected vs actual behavior
- Your OS and Bash version (`bash --version`)

## Code Style

- Keep scripts POSIX/Bash-friendly
- Prefer clear variable names (`principal`, `rate`, `time`)
- Add brief comments only when logic is non-obvious

## Pull Requests

- One logical change per PR
- Update `README.md` if behavior or usage changes
- Ensure `simple-interest.sh` still runs with sample inputs
